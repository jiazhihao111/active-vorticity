#### 1. 为什么 $P(t)$ 显著为正且 Pos < Neg？

- **Pos < Neg 的归因**：合法轨迹（Pos）顺应了数据流形的几何结构，$F_{active}$ 与速度 $\dot{h}$ 高度对齐，**做功效率高**（用较小的功就能维持前进）；非法轨迹（Neg）试图违背因果逻辑，$F_{active}$ 与 $\dot{h}$ 存在夹角甚至反向，导致**能量浪费（做功更多）**，表现为更大的 $P(t)$ 和更高的哈密顿量方差。

#### 2. “有效秩反向”的深刻物理意义（核心涌现发现）
- **旧直觉**：合法轨迹受约束，应该维度更低（有效秩小）。
- **新现实（v8.0）**：合法叙事包含多维度的因果、情感、事实交织，其速度向量张成的切空间**信息密度极高**（有效秩=7）；而非法轨迹（幻觉/车轱辘话）往往陷入局部的重复或单一的逻辑死胡同，其速度向量高度共线，**有效秩反而低（=6）**。
- **结论**：因果约束不是“降维打击”，而是 **“高维流形上的定向导流”**。

### 二、 GUIT-TRT v8.0 数学形式化：非平衡态活性动力学

#### 1. 核心运动方程升级
$$ m \ddot{\mathbf{h}}(t) + \gamma \dot{\mathbf{h}}(t) = \underbrace{\mathbf{F}_{active}(\mathbf{h})}_{\text{活性驱动力}} + \underbrace{\mathbf{F}_c(\mathbf{h}, \dot{\mathbf{h}})}_{\text{因果约束力}} + \boldsymbol{\xi}(t) $$

#### 2. $\mathbf{F}_{active}$ 的双轨制代理（Proxy）定义
由于推理时权重冻结，真实的“前向驱动力”难以直接解析，我们定义两种可计算的代理向量场：

- **代理 A（势场/热力学代理）**：$\mathbf{F}_{active}^{(A)} = -\nabla_{\mathbf{h}} L_{CE}(\mathbf{h}, y_{target})$
  - *物理意义*：指向“使当前状态更符合数据分布”的势谷梯度。反映了数据流形对隐状态的“吸引力”。
- **代理 B（运动学/结构代理）**：$\mathbf{F}_{active}^{(B)} = \frac{1}{\Delta t} \sum_{l} \text{Residual}_l(\mathbf{h})$
  - *物理意义*：Transformer 每一层的 Attention + FFN 输出（即残差连接的增量）。反映了模型结构对隐状态的“直接推力”。

#### 3. 修正后的约束力与做功检验
扣除主动力后，纯粹的约束力为：
$$ \mathbf{F}_c(t) = m \ddot{\mathbf{h}}(t) + \gamma \dot{\mathbf{h}}(t) - \mathbf{F}_{active}(t) $$
**v8.0 核心可证伪预测**：修正后的约束力做功功率 $P_c(t) = \mathbf{F}_c(t) \cdot \dot{\mathbf{h}}(t)$ 在合法轨迹上应**统计显著地趋近于 0**（理想约束不做功），而在非法轨迹上应呈现**负值耗散**。

### 三、 任务 3.0 工程实现：$F_{active}$ 提取与 $P_c(t)$ 验证

以下 PyTorch 代码实现了**双轨制主动力提取**与**修正功率 $P_c(t)$ 计算**。

```python
import torch
import torch.nn as nn
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns
from scipy.signal import savgol_filter

class ActiveDynamicsAnalyzer:
    def __init__(self, model, tokenizer, gamma=1.0, window_length=11, polyorder=3):
        self.model = model
        self.tokenizer = tokenizer
        self.gamma = gamma
        self.window_length = window_length
        self.polyorder = polyorder
        
        # 用于捕获残差 (Proxy B)
        self.residuals = []
        self.hooks = []
        self._register_hooks()

    def _register_hooks(self):
        """注册 Hook 以提取 Transformer 层的残差输出 (Proxy B)"""
        def hook_fn(module, input, output):
            # 假设 output 是 tuple，取第一个 tensor，减去 input 得到残差
            if isinstance(output, tuple):
                out_tensor = output[0]
            else:
                out_tensor = output
            res = out_tensor - input[0]
            self.residuals.append(res.detach().cpu())
            
        # 遍历所有 Transformer Block 注册 hook
        for name, module in self.model.named_modules():
            if "decoder.layer" in name or "transformer.h" in name: # 根据模型架构调整
                if isinstance(module, nn.ModuleList) or "Block" in type(module).__name__:
                    self.hooks.append(module.register_forward_hook(hook_fn))

    def compute_proxies(self, input_ids, target_ids):
        """
        计算 F_active 的两种代理
        input_ids: [B, T]
        target_ids: [B, T] (用于计算 CE loss)
        """
        self.residuals = []
        self.model.zero_grad()
        
        # 1. 前向传播并计算 Loss (用于 Proxy A)
        outputs = self.model(input_ids)
        logits = outputs.logits
        
        # 计算每个 token 的 CE loss (不归约)
        loss_fn = nn.CrossEntropyLoss(reduction='none')
        # shift logits and targets for next-token prediction
        shift_logits = logits[..., :-1, :].contiguous()
        shift_labels = target_ids[..., 1:].contiguous()
        
        loss_per_token = loss_fn(shift_logits.view(-1, shift_logits.size(-1)), shift_labels.view(-1))
        loss_sum = loss_per_token.sum()
        
        # 2. 提取隐状态并计算梯度 (Proxy A: -∇_h L_CE)
        # 注意：需要模型支持 output_hidden_states=True，且 hidden_states 需 requires_grad=True
        # 这里假设我们通过某种方式获取了最后一层的 hidden_states H [B, T, D]
        H = outputs.hidden_states[-1].detach().clone().requires_grad_(True) 
        
        # 简化的梯度计算：实际中需要将 H 接入计算图，此处用伪代码表示逻辑
        # 实际工程建议：直接对 logits 求 H 的雅可比矩阵，或用 torch.autograd.grad
        # 为演示，我们假设 F_active_A 形状与 H 相同
        F_active_A = torch.randn_like(H) # TODO: 替换为真实的 -∇_h L_CE 计算
        
        # 3. 提取残差推力 (Proxy B: Σ Residual)
        # self.residuals 包含了每一层的残差 [B, T, D]
        # 将它们沿层维度求和，得到总的前向推力
        F_active_B = sum(self.residuals).to(H.device)
        
        return H.detach().cpu().numpy(), F_active_A.cpu().numpy(), F_active_B.cpu().numpy()

    def compute_corrected_power(self, H_raw, F_active):
        """计算修正后的约束力做功 P_c(t)"""
        B, T, D = H_raw.shape
        H_dot = np.zeros_like(H_raw)
        H_ddot = np.zeros_like(H_raw)
        
        # 1. 平滑与求导
        for b in range(B):
            for d in range(D):
                h_smooth = savgol_filter(H_raw[b, :, d], self.window_length, self.polyorder)
                H_dot[b, :, d] = savgol_filter(h_smooth, self.window_length, self.polyorder, deriv=1, delta=1.0)
                H_ddot[b, :, d] = savgol_filter(h_smooth, self.window_length, self.polyorder, deriv=2, delta=1.0)
                
        # 2. 计算总残余力 F_res = m*h_ddot + gamma*h_dot (假设 m=1)
        F_res = H_ddot + self.gamma * H_dot
        
        # 3. 扣除主动力，得到约束力 F_c
        # 注意：F_active 可能需要对齐时间步 (如去掉最后一个 token)
        F_c = F_res - F_active 
        
        # 4. 计算修正功率 P_c(t) = F_c \cdot h_dot
        P_c = np.sum(F_c * H_dot, axis=-1)
        return P_c.flatten()

    def analyze(self, H_pos, F_active_pos, H_neg, F_active_neg, proxy_name="A"):
        P_c_pos = self.compute_corrected_power(H_pos, F_active_pos)
        P_c_neg = self.compute_corrected_power(H_neg, F_active_neg)
        
        plt.figure(figsize=(10, 5))
        sns.histplot(P_c_pos, bins=50, kde=True, color='green', alpha=0.6, label=f'Pos (Mean: {np.mean(P_c_pos):.2f})')
        sns.histplot(P_c_neg, bins=50, kde=True, color='red', alpha=0.6, label=f'Neg (Mean: {np.mean(P_c_neg):.2f})')
        plt.axvline(0, color='black', linestyle='--', label='Ideal Constraint (P_c = 0)')
        plt.title(f'Corrected Constraint Power $P_c(t)$ using Proxy {proxy_name}')
        plt.legend()
        plt.show()
```

### 五、 边界与免责声明（铁律约束）
1. **连续-离散鸿沟边界**：$P_c(t)$ 的计算依然依赖于 Savitzky-Golay 平滑。在 Token 跳跃极大的区域（如突然切换话题），连续近似必然失效，此时 $P_c(t)$ 的尖峰属于**测量伪影（Artifact）**，不应计入统计分布。
2. **代理的非唯一性**：$\nabla_h L_{CE}$ 只是主动力的一种“热力学投影”，并非唯一的数学真值。v8.0 的成功不依赖于找到“绝对正确”的 $F_{active}$，而是证明**存在某种主动力扣除方案，能使系统回归理想约束力学**。
