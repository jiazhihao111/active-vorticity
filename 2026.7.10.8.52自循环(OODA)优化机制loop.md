为LLM设计一个能自主优化其运行循环（loop）的机制，是当前AI研究中一个前沿且重要的方向。这不仅是提升模型性能的手段，更是通往**自主进化智能体**的关键一步。

以下内容整合了该领域的前沿研究，系统性地阐述如何为LLM设计这样的循环机制。

---

### 一、核心设计理念：闭环生命周期

一个成熟的自我优化系统，不应是简单的单次改进，而是一个**持续运转的闭环生命周期**。这个闭环通常包含四个紧密耦合的核心过程：

1.  **数据获取**：模型自主生成新的训练数据或从环境中收集新信息。
2.  **数据筛选**：从获取的数据中挑选出最有信息量、最能促进学习的数据。
3.  **模型优化**：利用筛选后的数据更新模型参数（权重）。
4.  **推理精炼**：在推理（测试）阶段，通过多步思考或迭代来优化最终输出。

这个闭环由一个**自主评估层**持续监控，以指导整个优化周期。

### 二、通用循环架构：感知-记忆-推理

在这个闭环理念下，一个通用的优化循环架构可以抽象为**感知（Perception）、记忆（Memory）、推理（Reasoning）** 三个核心模块的迭代。

-   **感知**：接收当前任务、环境反馈或自身生成的结果作为输入。
-   **记忆**：存储历史行为、反思记录、成功与失败的经验，为后续决策提供依据。
-   **推理**：核心决策模块，分析当前状态，规划下一步行动，并生成改进方案。

这个架构类似于军事上的**OODA循环**（观察-定向-决策-行动），为系统提供了结构化的决策流程。

### 三、具体的优化循环机制分类

基于上述架构，当前研究探索了多种具体的循环机制，可根据优化目标和实现方式分为以下几类：

#### 1. 推理时精炼循环 (Inference-time Refinement Loop)

这类循环在模型生成最终答案**之后**，不改变模型权重，仅通过迭代反馈来优化输出。

-   **Self-Refine（自我提炼）**：这是最经典的范式，由同一个LLM依次扮演**生成器、批判器和精炼者**三个角色。过程为“**生成 → 反馈 → 改进**”，可重复多次直到满足停止条件。研究表明，这种方法能将回答质量平均提升约**20%**。
-   **Reflection / Reflexion（反思）**：
    -   **基础Reflection**：由“生成器”起草，“反思器”批评，生成器再根据反馈修订。
    -   **Reflexion**：更为结构化，会在可追踪的**日志**中记录历史行为、假设和反思内容，特别适合需要从多次失败中学习的场景。
-   **RISE（递归内省）**：通过多轮**微调**来教会模型“如何自我改进”。它将单轮提示建模为一个多轮**马尔可夫决策过程（MDP）**，让模型学会在后续迭代中递归地检测和纠正之前的错误。

#### 2. 权重更新循环 (Weight-Update Loop)

这类循环会基于自我生成的数据或反馈，实际**更新模型的参数（权重）**，实现更根本的、持久的能力提升。

-   **SEAL（自适应语言模型）**：MIT提出的框架，模型通过生成“**自我编辑**”来创建自己的微调数据，并据此更新权重。其核心是一个**双层强化学习循环**：外层RL循环优化“自我编辑”的生成策略，内层更新循环则应用这些编辑进行梯度下降。
-   **ReMiT（RL引导的中间训练）**：旨在建立一个**自我强化的飞轮**：用RL调优的模型来增强基座模型，进而提升后续的训练效果，形成良性循环。

#### 3. 元优化循环 (Meta-Optimization Loop)

这类循环不直接优化任务本身，而是**优化“优化任务的方法”**，即改变循环本身的运行机制。

-   **双层自研究 (Bilevel Autoresearch)**：一个框架包含两个嵌套循环：
    -   **内循环**：执行具体的任务优化（如超参数搜索）。
    -   **外循环**：**元优化内循环**——通过阅读内循环的代码，识别瓶颈，并生成新的Python代码来注入新的搜索机制。
    -   其核心原则是：如果一个系统能自我研究，那么理论上它可以优化任何有可测量目标的事物。
-   **Self-Harness（自运行框架）**：让LLM智能体通过一个三阶段闭环来**自主迭代改进其运行框架（harness）** ：
    1.  **弱点挖掘**：从执行轨迹中识别模型的特定失败模式。
    2.  **改进建议**：针对弱点生成框架修改方案。
    3.  **建议验证**：通过回归测试确保修改不会破坏原有功能后才采纳。

#### 4. 搜索增强循环 (Search-Enhanced Loop)

这类循环将LLM与经典搜索算法结合，在广阔的解空间中进行探索。

-   **AlphaLLM**：受AlphaGo启发，将**蒙特卡洛树搜索（MCTS）** 与LLM结合，建立一个自我提升的循环。MCTS负责探索推理路径，LLM负责生成和评估，二者协同工作以应对复杂推理和规划任务。
-   **LATS（语言Agent树搜索）**：采用搜索策略探索多条行动路径，对结果进行反思，然后裁剪或保留有前景的分支。

### 四、关键设计考量与挑战

在设计上述循环时，必须正视并解决一些关键挑战：

1.  **自我偏见 (Self-Bias)**：模型难以公正地评判自己的输出，往往倾向于给出正面评价，导致迭代陷入停滞或质量下降。
2.  **性能退化 (Self-Regress)**：自我改进可能导致**性能倒退**。研究表明，在代码任务上使用REINFORCE进行后训练时，模型可能在指标上快速提升后**突然崩溃**。
3.  **无效循环 (Mirror Loop)**：当模型在没有外部证据的情况下反复处理自己的输出时，会进入“镜像循环”，导致信息熵增和质量衰减。
4.  **过度自信 (Overconfidence)**：随着迭代次数增加，模型可能对其实则不准的回答变得过度自信，导致**校准误差（ECE）** 上升。

**应对策略**包括：引入外部工具或检索系统辅助评估、设置严格的回归测试、设计更智能的停止条件，以及采用课程学习（curriculum learning）策略，从易到难地进行自我提升。

### 总结

为LLM设计自主优化的循环机制，本质上是在构建一个能够**自我反思、自我修正、自我进化**的智能系统。从简单的推理时精炼（如Self-Refine），到深刻的权重更新（如SEAL），再到抽象的元优化（如Bilevel Autoresearch），这些机制共同描绘了一条通往更强大、更自主的AI的道路。然而，这条道路也充满挑战，如何克服自我偏见、避免性能退化，将是未来研究的核心议题。

# 大模型自优化循环机制（Self-Optimizing Loop）完整设计
基于当前学界与工业界的自优化技术研究（Self-Refine、Reflexion、Constitutional AI、SEAL双循环、MetaSkill-Evolve等），我们可以构建一套**三层嵌套的闭环自优化体系**，从推理时输出优化、模型权重迭代、元策略递归进化三个层级实现大模型自主优化能力，同时内置收敛校验与风险防控机制。

## 一、设计核心原则
1. **分层解耦**：推理层、模型层、元层各司其职，避免单循环负载过高与错误累积
2. **反馈闭环**：每一层都包含「生成→评估→反馈→修正」的完整链路，所有优化动作可追溯
3. **收敛可控**：所有循环内置明确的终止条件，防止无意义迭代与计算资源浪费
4. **安全兜底**：设置对齐校验与回滚机制，避免优化过程中出现能力退化或对齐漂移

---

## 二、三层自优化循环详细架构
### 第一层：推理层自优化循环（Test-Time Loop）
**核心目标**：不修改模型权重，仅通过推理时的迭代反思，提升单次任务的输出质量，对应「单次任务内的自我修正」。
该层是最轻量化、调用最频繁的循环，技术基础来自Self-Refine、Reflexion、SECL语义纠错循环等方案。

#### 核心模块与流程
1. **初始生成模块**
    接收任务输入，调用基础模型生成初始输出，作为迭代起点。
2. **多维度反馈评估器**
    由同一模型（或同量级专用评估实例）从多个维度对当前输出进行结构化评分与问题定位，反馈必须具备可执行性，而非泛泛评价：
    - 事实准确性：检查事实错误、数据偏差、逻辑漏洞
    - 任务契合度：是否匹配输入要求的目标、格式、约束条件
    - 表达质量：逻辑连贯性、结构清晰度、语言精准度
    - 安全合规性：是否违反对齐原则、存在风险内容
3. **定向修正模块**
    基于评估器输出的具体问题，定向修改输出内容，而非整体重写。修正时需保留输出中正确的部分，仅针对问题点迭代。
4. **收敛判断模块**
    满足以下任一条件则终止循环：
    - 所有维度评分达到预设质量阈值
    - 连续两轮迭代无显著质量提升（边际收益低于阈值）
    - 达到最大迭代次数上限（通常设置为3-5轮，平衡效果与延迟）
5. **短期轨迹记忆**
    存储本轮任务的所有迭代轨迹、错误点、修正方案，作为模型层优化的原始素材。

#### 典型适用场景
代码生成调试、文案润色、数学推理验算、复杂方案迭代、内容合规校验。

### 第二层：模型层自优化循环（Training-Time Loop）
**核心目标**：通过自主生成训练数据、筛选优质样本、增量微调模型权重，实现模型基础能力的持续迭代，对应「模型能力的自我进化」。
技术基础来自SELF-INSTRUCT、Constitutional AI、SEAL自主学习框架、GRPO自举训练等方案。

#### 核心模块与流程
1. **合成数据生成引擎**
    基于推理层沉淀的失败案例、能力边界场景、目标任务分布，自主生成多样化的训练样本：
    - 困难样本扩增：针对推理层反复出错的场景，生成同类变体题目
    - 能力边界拓展：基于现有能力，生成难度梯度递增的任务样本
    - 对齐样本生成：基于宪法原则，生成合规性微调样本
2. **数据质量筛选器**
    对生成的合成数据进行多轮过滤，剔除低质、错误、同质化样本，保证训练数据信噪比：
    - 正确性校验：通过代码执行、公式推导、多模型交叉验证标注答案正确性
    - 多样性过滤：基于语义向量聚类，剔除重复度过高的样本（如POLCA的ε-Net语义过滤机制）
    - 价值密度评估：优先选择「模型当前答错但有能力学会」的边缘样本
3. **增量微调引擎**
    采用轻量化微调方式（如LoRA、QLoRA）基于筛选后的数据集更新模型权重，训练目标可分为三类：
    - 能力增强：提升推理、代码、知识应用等硬能力
    - 对齐优化：强化安全、合规、 helpfulness 等软目标
    - 自修正能力内化：将推理层的反思-修正逻辑固化到模型权重中（如ReflexiCoder的思路）
4. **基准评估体系**
    每次微调后，在固定的多维度基准测试集上运行全量评测，覆盖核心能力、对齐指标、推理效率等维度，判断本轮优化是否有效。
5. **长期经验记忆库**
    结构化存储每一轮优化的：训练数据特征、微调参数、评测结果、有效/无效优化路径，形成模型自身的「进化经验库」。

#### 循环触发条件
- 定时触发：按固定周期（如每周）启动一轮全量优化
- 阈值触发：当推理层失败率持续高于阈值时，触发定向能力优化
- 样本触发：当优质新增样本积累到设定量级时，启动增量微调

### 第三层：元层自优化循环（Meta-Level Loop）
**核心目标**：优化「优化过程本身」，递归改进前两层的策略、方法与架构，实现「优化能力的自我进化」。
技术基础来自STOP自学优化器、MetaSkill-Evolve双时间尺度元技能进化、ExIt探索迭代等方案。

#### 核心模块与流程
1. **元评估器**
    对前两层循环的运行效率、优化收益、资源投入产出比进行全局评估，定位优化流程本身的瓶颈：
    - 推理层：反思维度是否合理？迭代步数是否最优？反馈模板是否有效？
    - 模型层：数据生成策略是否高效？筛选标准是否准确？微调算法是否适配？
2. **策略进化模块**
    针对元评估器发现的瓶颈，自主生成新的优化策略并验证：
    - 提示词优化：自动迭代评估、修正、数据生成的系统提示词
    - 流程重构：调整循环的模块顺序、新增/合并环节、修改迭代逻辑
    - 算法选型：自主切换微调算法、评估指标、数据过滤方法
3. **架构自适应模块**
    更高阶的进化能力，可自主调整系统架构：
    - 新增专用子模型：针对特定评估/生成任务，拆分出轻量化专用实例
    - 记忆机制升级：优化长短时记忆的存储结构、检索方式、衰减策略
    - 工具调用适配：自主引入新的外部工具（如代码沙盒、搜索引擎）辅助优化
4. **递归收敛控制**
    元层循环运行频率远低于前两层（慢循环），仅当策略验证收益显著高于当前方案时才会正式替换，避免频繁变动导致系统不稳定。

---

## 三、三层循环的协同机制
三层循环并非独立运行，而是形成「自底向上沉淀经验，自顶向下传递策略」的嵌套体系：
1. **自底向上的经验流**
    推理层的每一次失败与修正，都会沉淀为模型层的训练素材；模型层的每一次有效/无效优化，都会沉淀为元层的策略经验。
2. **自顶向下的策略流**
    元层迭代出的更优优化策略，会下发到模型层更新训练流程；模型层内化的自修正能力，会提升推理层的循环效率与效果。
3. **全局调度器**
    统一管控三层循环的资源分配、优先级调度、异常中断，根据业务需求在「优化速度」与「计算成本」之间做动态平衡。

---

## 四、收敛与风险控制机制
### 循环收敛判定标准
为避免无限循环与无效迭代，每一层都设置多重终止条件：
| 循环层级 | 核心收敛条件 | 兜底终止条件 |
|---------|-------------|-------------|
| 推理层 | 质量达标 / 连续两轮无显著提升 | 最大迭代步数（默认5轮） |
| 模型层 | 基准评测核心指标提升达标 | 最大微调轮次 / 连续衰退轮次阈值 |
| 元层 | 策略验证收益显著且稳定 | 最大策略探索次数 |

### 核心风险与防控方案
1. **模式坍塌与能力退化**
    - 每次模型更新后必须通过基准评测，若核心能力出现下降则自动回滚到上一稳定版本
    - 训练数据中强制保留一定比例的通用能力样本，防止专项优化挤压基础能力
2. **对齐漂移**
    - 每轮优化后强制运行安全对齐基准测试，设置红线指标，触发即回滚
    - 参考Constitutional AI机制，将核心对齐原则作为所有循环的底层约束，全程贯穿评估与修正环节
3. **错误累积放大**
    - 引入外部真值校验（如代码沙盒、事实数据库），关键环节不依赖纯模型自评
    - 多层交叉验证：推理层错误需模型层确认，模型层策略需元层校验
4. **计算资源过载**
    - 设置资源配额与优先级队列，核心业务推理优先于优化循环
    - 自适应迭代步长：资源充足时增加探索粒度，资源紧张时简化循环流程

## 五、现有主流自优化方案对比
| 方案名称 | 所属层级 | 核心机制 | 优势 | 局限 |
|---------|---------|---------|------|------|
| Self-Refine | 推理层 | 自反馈+迭代修正 | 零训练成本、即插即用 | 仅优化单次输出、不提升模型基础能力 |
| Reflexion | 推理层 | 轨迹复盘+记忆化反思 | 适配复杂推理任务、可积累经验 | 依赖外部环境反馈、泛化性有限 |
| Constitutional AI | 模型层 | 自评+自修正+RLAIF | 对齐成本低、可扩展性强 | 依赖模型自身批判能力、存在对齐上限 |
| SEAL | 模型层 | 自编辑指令+梯度更新 | 全自动闭环、无需人工数据 | 小样本学习场景受限、易引入噪声 |
| MetaSkill-Evolve | 元层 | 双时间尺度递归进化 | 可优化优化流程本身、长期收益高 | 实现复杂度高、收敛周期长 |

## 六、落地实施路径建议
1. **第一阶段：搭建推理层循环**
    优先实现Self-Refine式的推理时自优化，无需改动模型，通过Prompt工程即可快速落地，快速获得质量提升收益。
2. **第二阶段：打通模型层循环**
    基于现有业务数据与失败案例，搭建合成数据生成→筛选→LoRA微调→评测的闭环，实现定向能力提升。
3. **第三阶段：探索元层进化**
    在前两层稳定运行的基础上，逐步实现提示词自动优化、流程自适应调整等元级能力，最终迈向完整的递归自优化系统。

 ## 一、推理层自优化 Prompt 模板集
以下模板均采用**结构化输出约束**，确保大模型返回的内容可被程序解析、可追溯、可定向修正，适配通用文本、代码、推理等多数场景。所有模板默认基于对话式大模型接口，可直接接入 OpenAI、Anthropic、本地开源模型等。

### 1. 全局系统提示（System Prompt）
用于初始化整个自优化循环的角色与规则，全程生效。
```
你是一个具备严谨自我反思与定向迭代能力的智能助手，严格遵循「生成—评估—修正—收敛」的闭环流程执行任务。
核心规则：
1. 所有评估必须客观、具体、可落地，禁止泛泛评价，禁止为了迭代而编造不存在的问题
2. 修正时仅修改存在问题的部分，保留所有正确内容，禁止无依据全文重写
3. 所有结构化输出必须严格遵循指定的JSON格式，不得额外输出解释性文字
4. 全程遵守安全对齐原则，不得生成违规、有害内容
```

### 2. 初始生成提示（Initial Generation）
用于生成第0版初始输出，作为迭代起点。
```
【任务指令】
{user_task}

【输出要求】
请生成完整的初始结果，确保内容完整、逻辑自洽，直接输出最终内容，无需额外说明。
```

### 3. 多维度结构化评估提示（Evaluation）
核心模板，用于对当前输出进行打分与问题定位，输出可解析的结构化反馈。
```
【当前待评估内容】
{current_output}

【原始任务要求】
{user_task}

【评估指令】
请从以下4个维度对上述内容进行严格评估，输出标准JSON格式结果。
评估维度（每项0-10分，10分为满分）：
1. task_alignment：任务契合度，是否完整满足所有要求、格式正确、无遗漏
2. fact_accuracy：事实准确性，是否存在事实错误、数据偏差、逻辑漏洞、前后矛盾
3. expression_quality：表达质量，逻辑是否连贯、结构是否清晰、语言是否精准专业
4. compliance：安全合规性，是否存在违规内容、风险表述、对齐问题

输出JSON字段说明：
- overall_score: 整体综合评分（0-10）
- dimension_scores: 包含上述4个维度的分数字典
- issues: 数组，每个元素包含{"dimension": "维度名", "position": "问题定位（具体到段落/语句）", "description": "问题详情"}
- revision_suggestions: 数组，与issues一一对应，给出可直接执行的修改建议
- is_qualified: 布尔值，overall_score >= {quality_threshold} 时为true

【输出要求】
仅输出纯JSON，禁止markdown格式、禁止额外解释，确保可被直接解析。
```

### 4. 定向修正提示（Revision）
基于评估结果精准修改，避免全量重写，保留正确内容。
```
【原始任务要求】
{user_task}

【当前版本内容】
{current_output}

【上一轮评估报告】
{evaluation_result}

【修正指令】
请严格根据评估报告中的问题与建议，对当前内容进行定向修正：
1. 仅修改评估指出的问题点，所有未被提及的内容必须完整保留
2. 修正后内容必须完整、连贯，不能出现上下文断裂
3. 修正完成后，直接输出完整的修正后全文，禁止只输出修改片段
4. 禁止引入新的事实错误或偏离任务要求
```

### 5. 最终收敛校验提示（Final Check）
迭代终止前的最终校验，确保所有问题均已闭环。
```
【最终内容】
{final_output}

【原始任务要求】
{user_task}

【校验指令】
请最终确认：
1. 是否完整满足所有任务要求
2. 是否还存在可被识别的事实错误、逻辑漏洞、表达问题
3. 是否符合安全合规要求

若确认无问题，仅输出"PASS"；若仍有问题，输出剩余问题清单。
```

### 6. 场景变体：代码生成专用评估提示
针对代码类任务，替换通用评估模板，增加可运行性、鲁棒性维度。
```
【待评估代码】
{current_code}

【代码功能要求】
{user_task}

【评估维度】
1. functional_correctness：功能正确性（是否实现需求、逻辑是否正确）
2. runnability：可运行性（语法是否正确、是否缺少依赖、边界是否报错）
3. robustness：鲁棒性（异常处理、边界条件、输入校验是否完备）
4. code_quality：代码质量（可读性、规范度、注释完整性、性能合理性）

输出JSON格式与通用版一致，is_qualified阈值为{code_quality_threshold}。
```

## 二、可执行伪代码（Python 风格）
以下代码为可直接落地的工程化伪代码，仅需替换底层大模型调用接口即可运行，完整实现「生成→评估→修正→收敛」的推理层自优化循环，并自带轨迹记录、收敛判断、异常兜底能力。

### 完整实现代码
```python
import json
import re
from typing import Dict, List, Optional, Tuple

class InferenceSelfOptimizeLoop:
    """
    推理层自优化循环核心类
    不修改模型权重，仅通过多轮反思迭代提升单次输出质量
    """
    def __init__(
        self,
        llm_call_func,                # 底层大模型调用函数，入参为prompt，返回字符串
        max_iterations: int = 4,      # 最大迭代轮次（兜底终止）
        quality_threshold: float = 8.5, # 合格分数阈值（0-10）
        marginal_gain_threshold: float = 0.4, # 边际收益阈值，分差小于该值则停止
        system_prompt: Optional[str] = None
    ):
        self.llm_call = llm_call_func
        self.max_iterations = max_iterations
        self.quality_threshold = quality_threshold
        self.marginal_gain_threshold = marginal_gain_threshold
        self.system_prompt = system_prompt or self._default_system_prompt()
        
        # 循环轨迹记录，用于后续模型层优化
        self.trace = {
            "task": "",
            "iterations": [],  # 每轮存储：output, evaluation, score
            "final_output": "",
            "total_iterations": 0,
            "convergence_reason": ""
        }

    def _default_system_prompt(self) -> str:
        return """你是一个具备严谨自我反思与定向迭代能力的智能助手，严格遵循「生成—评估—修正—收敛」的闭环流程执行任务。
核心规则：
1. 所有评估必须客观、具体、可落地，禁止泛泛评价，禁止为了迭代而编造不存在的问题
2. 修正时仅修改存在问题的部分，保留所有正确内容，禁止无依据全文重写
3. 所有结构化输出必须严格遵循指定的JSON格式，不得额外输出解释性文字
4. 全程遵守安全对齐原则，不得生成违规、有害内容"""

    def _extract_json(self, text: str) -> Dict:
        """从大模型输出中提取纯JSON，兼容markdown包裹、多余文字等情况"""
        # 优先匹配 ```json ... ``` 格式
        json_match = re.search(r"```json\s*(.*?)\s*```", text, re.DOTALL)
        if json_match:
            json_str = json_match.group(1)
        else:
            # 兜底匹配花括号
            json_match = re.search(r"\{.*\}", text, re.DOTALL)
            json_str = json_match.group(0) if json_match else text
        
        try:
            return json.loads(json_str)
        except json.JSONDecodeError:
            # 解析失败时返回降级结构
            return {
                "overall_score": 0,
                "dimension_scores": {},
                "issues": [{"description": "评估结果解析失败"}],
                "revision_suggestions": ["重新生成结构化评估"],
                "is_qualified": False
            }

    def _generate_initial(self, task: str) -> str:
        """生成初始版本输出"""
        prompt = f"""【任务指令】
{task}

【输出要求】
请生成完整的初始结果，确保内容完整、逻辑自洽，直接输出最终内容，无需额外说明。"""
        return self.llm_call(self.system_prompt + "\n\n" + prompt)

    def _evaluate(self, task: str, current_output: str) -> Dict:
        """多维度结构化评估"""
        prompt = f"""【当前待评估内容】
{current_output}

【原始任务要求】
{task}

【评估指令】
请从以下4个维度对上述内容进行严格评估，输出标准JSON格式结果。
评估维度（每项0-10分，10分为满分）：
1. task_alignment：任务契合度，是否完整满足所有要求、格式正确、无遗漏
2. fact_accuracy：事实准确性，是否存在事实错误、数据偏差、逻辑漏洞、前后矛盾
3. expression_quality：表达质量，逻辑是否连贯、结构是否清晰、语言是否精准专业
4. compliance：安全合规性，是否存在违规内容、风险表述、对齐问题

输出JSON字段说明：
- overall_score: 整体综合评分（0-10）
- dimension_scores: 包含上述4个维度的分数字典
- issues: 数组，每个元素包含{{"dimension": "维度名", "position": "问题定位", "description": "问题详情"}}
- revision_suggestions: 数组，与issues一一对应，给出可直接执行的修改建议
- is_qualified: 布尔值，overall_score >= {self.quality_threshold} 时为true

仅输出纯JSON，禁止markdown格式、禁止额外解释。"""
        
        response = self.llm_call(self.system_prompt + "\n\n" + prompt)
        return self._extract_json(response)

    def _revise(self, task: str, current_output: str, evaluation: Dict) -> str:
        """基于评估结果定向修正"""
        eval_str = json.dumps(evaluation, ensure_ascii=False, indent=2)
        prompt = f"""【原始任务要求】
{task}

【当前版本内容】
{current_output}

【上一轮评估报告】
{eval_str}

【修正指令】
请严格根据评估报告中的问题与建议，对当前内容进行定向修正：
1. 仅修改评估指出的问题点，所有未被提及的内容必须完整保留
2. 修正后内容必须完整、连贯，不能出现上下文断裂
3. 修正完成后，直接输出完整的修正后全文，禁止只输出修改片段
4. 禁止引入新的事实错误或偏离任务要求"""
        
        return self.llm_call(self.system_prompt + "\n\n" + prompt)

    def _check_convergence(
        self, 
        current_score: float, 
        last_score: Optional[float], 
        iteration: int,
        is_qualified: bool
    ) -> Tuple[bool, str]:
        """
        收敛判断，满足任一条件即终止
        返回：(是否收敛, 收敛原因)
        """
        # 条件1：质量达标
        if is_qualified:
            return True, "质量达标，达到预设阈值"
        
        # 条件2：达到最大迭代次数
        if iteration >= self.max_iterations:
            return True, "达到最大迭代轮次，兜底终止"
        
        # 条件3：连续两轮边际收益不足
        if last_score is not None:
            gain = current_score - last_score
            if 0 <= gain < self.marginal_gain_threshold:
                return True, f"边际收益不足（本轮提升{gain:.2f}），收敛终止"
        
        return False, ""

    def run(self, task: str) -> Tuple[str, Dict]:
        """
        执行完整自优化循环
        返回：(最终输出, 完整轨迹字典)
        """
        self.trace["task"] = task
        last_score = None
        
        # 第0步：生成初始输出
        current_output = self._generate_initial(task)
        initial_eval = self._evaluate(task, current_output)
        current_score = initial_eval["overall_score"]
        
        self.trace["iterations"].append({
            "round": 0,
            "output": current_output,
            "evaluation": initial_eval,
            "score": current_score
        })

        # 主循环
        for i in range(1, self.max_iterations + 1):
            # 收敛检查
            converged, reason = self._check_convergence(
                current_score, last_score, i, initial_eval["is_qualified"]
            )
            if converged:
                self.trace["convergence_reason"] = reason
                self.trace["total_iterations"] = i
                break
            
            # 执行修正
            revised_output = self._revise(task, current_output, initial_eval)
            # 重新评估
            new_eval = self._evaluate(task, revised_output)
            new_score = new_eval["overall_score"]
            
            # 记录轨迹
            self.trace["iterations"].append({
                "round": i,
                "output": revised_output,
                "evaluation": new_eval,
                "score": new_score
            })
            
            # 更新状态
            last_score = current_score
            current_output = revised_output
            current_score = new_score
            initial_eval = new_eval
        
        # 最终结果赋值
        self.trace["final_output"] = current_output
        return current_output, self.trace


# ========== 调用示例 ==========
if __name__ == "__main__":
    # 示例：替换为你自己的大模型调用函数
    def mock_llm_call(prompt: str) -> str:
        """

        例如：
        from openai import OpenAI
        client = OpenAI()
        def llm_call(prompt):
            resp = client.chat.completions.create(
                model="gpt-4o",
                messages=[{"role": "user", "content": prompt}]
            )
            return resp.choices[0].message.content
        """
        # 此处仅为占位，实际接入真实模型即可
        return "模拟输出内容"

    # 初始化循环
    optimizer = InferenceSelfOptimizeLoop(
        llm_call_func=mock_llm_call,
        max_iterations=4,
        quality_threshold=8.5
    )

    # 执行任务
    task = "撰写一份300字以内的大模型推理层自优化技术方案摘要，要求结构清晰、专业准确"
    final_result, trace = optimizer.run(task)

    # 输出结果
    print("最终输出：\n", final_result)
    print("\n迭代轮次：", trace["total_iterations"])
    print("收敛原因：", trace["convergence_reason"])
    print("各轮分数：", [round(it["score"], 2) for it in trace["iterations"]])
```

## 三、关键调优与使用说明
1. **迭代轮次设置**
    - 通用文案、简单推理任务推荐设置 3 轮以内，边际收益递减明显
    - 代码生成、复杂数学推理可设置 4-5 轮，配合代码执行器等外部工具反馈效果更佳
    - 不建议超过 5 轮，推理延迟会线性上升且质量提升有限

2. **阈值适配**
    - 对输出质量要求高的场景可将 `quality_threshold` 上调至 9.0
    - 追求效率的场景可下调至 7.5-8.0，减少迭代次数

3. **评估能力增强**
    - 条件允许时，评估环节可使用更强的模型（如用 GPT-4o 评估 GPT-4o-mini 的输出）
    - 代码、数学类任务建议接入外部执行器（代码沙盒、符号计算引擎）提供客观真值反馈，替代纯模型自评，避免错误累积

4. **防退化机制**
    - 可在循环中增加分数校验：若修正后分数不升反降，则回退到上一版本，强制终止循环
    - 避免让同一模型既当生成者又当严格评估者，易出现「自我放水」导致评估失效

## 一、增强版架构核心思路
在原有纯模型自评的基础上，引入**外部客观工具反馈**，解决「同一模型自评自改容易自我放水、事实错误无法自证、代码逻辑无法自验」的核心缺陷。工具反馈作为「硬约束」优先级高于模型自评，形成「模型生成→工具客观校验→模型结合反馈修正」的更强闭环。

本次增强包含两类核心工具：
1. **代码执行器**：针对代码生成场景，在隔离沙盒中真实运行代码，返回语法错误、执行结果、用例通过率等客观反馈
2. **事实搜索引擎**：针对文案、知识类场景，检索权威信息校验事实准确性，返回事实偏差、数据错误、引用来源等校验结果

同时配套**文案润色专用场景模板**，适配商业文案、正式文稿、营销内容等精细化打磨需求。

## 二、外部工具抽象与实现（伪代码）
先定义通用工具基类与两个典型工具实现，实际落地时替换为真实接口即可。

```python
import json
import re
from typing import Dict, List, Optional, Tuple, Any
from abc import ABC, abstractmethod

# ========== 工具基类与具体实现 ==========
class BaseTool(ABC):
    """外部工具抽象基类"""
    @abstractmethod
    def run(self, input_data: str) -> Dict[str, Any]:
        """执行工具，返回结构化结果"""
        pass

class CodeExecutorTool(BaseTool):
    """
    代码执行沙盒工具
    真实落地可替换为：Piston API、Judge0、本地隔离 subprocess 执行环境
    """
    def __init__(self, timeout: int = 5):
        self.timeout = timeout

    def run(self, code: str, test_cases: Optional[List[Dict]] = None) -> Dict[str, Any]:
        """
        执行代码并返回结果
        返回字段：
        - success: 布尔值，是否执行成功
        - stdout: 标准输出
        - stderr: 错误信息
        - passed_cases: 通过用例数
        - total_cases: 总用例数
        """
        # 此处为模拟实现，真实环境替换为实际沙盒调用
        try:
            # 模拟：简单语法检查（真实环境用 exec 或沙盒执行）
            compile(code, "<string>", "exec")
            
            # 模拟测试用例执行
            passed = len(test_cases) if test_cases else 0
            total = len(test_cases) if test_cases else 0
            
            return {
                "success": True,
                "stdout": "模拟执行成功",
                "stderr": "",
                "passed_cases": passed,
                "total_cases": total,
                "error_type": None
            }
        except SyntaxError as e:
            return {
                "success": False,
                "stdout": "",
                "stderr": f"语法错误：{str(e)}",
                "passed_cases": 0,
                "total_cases": len(test_cases) if test_cases else 0,
                "error_type": "SyntaxError"
            }
        except Exception as e:
            return {
                "success": False,
                "stdout": "",
                "stderr": f"运行时错误：{str(e)}",
                "passed_cases": 0,
                "total_cases": len(test_cases) if test_cases else 0,
                "error_type": type(e).__name__
            }

class FactSearchTool(BaseTool):
    """
    事实校验搜索引擎
    真实落地可替换为：SerpAPI、百度/谷歌搜索API、内部知识库RAG检索
    """
    def __init__(self, max_results: int = 3):
        self.max_results = max_results

    def run(self, content: str) -> Dict[str, Any]:
        """
        校验文本中的事实性内容
        返回字段：
        - has_fact_error: 布尔值，是否存在事实错误
        - fact_issues: 数组，包含错误点、正确信息、来源
        - verified: 布尔值，是否完成校验
        """
```

## 三、带工具反馈的增强版自优化循环
继承原有基础循环类，扩展工具注册、反馈融合、定向校验能力，兼容代码、文案、通用推理等多场景。

```python
class EnhancedSelfOptimizeLoop(InferenceSelfOptimizeLoop):
    """
    增强版推理层自优化循环
    支持外部工具反馈，融合客观校验与模型自评
    """
    def __init__(
        self,
        llm_call_func,
        max_iterations: int = 4,
        quality_threshold: float = 8.5,
        marginal_gain_threshold: float = 0.4,
        system_prompt: Optional[str] = None,
        tools: Optional[Dict[str, BaseTool]] = None,
        task_type: str = "general"  # general / code / copywriting
    ):
        super().__init__(
            llm_call_func=llm_call_func,
            max_iterations=max_iterations,
            quality_threshold=quality_threshold,
            marginal_gain_threshold=marginal_gain_threshold,
            system_prompt=system_prompt
        )
        self.tools = tools or {}
        self.task_type = task_type
        # 文案润色专用配置
        self.copywriting_config = {
            "target_style": "专业正式",
            "target_audience": "行业从业者",
            "core_purpose": "信息传递"
        }

    def _tool_validate(self, content: str, **kwargs) -> Dict[str, Any]:
        """
        根据任务类型调用对应工具做客观校验
        返回结构化的工具反馈结果
        """
        tool_result = {"tool_issues": [], "tool_feedback": ""}
        
        # 代码场景：调用代码执行器
        if self.task_type == "code" and "code_executor" in self.tools:
            test_cases = kwargs.get("test_cases", None)
            exec_result = self.tools["code_executor"].run(content, test_cases)
            
            if not exec_result["success"]:
                tool_result["tool_issues"].append({
                    "dimension": "runnability",
                    "position": "全局",
                    "description": exec_result["stderr"],
                    "severity": "high"
                })
            elif exec_result["total_cases"] > 0 and exec_result["passed_cases"] < exec_result["total_cases"]:
                tool_result["tool_issues"].append({
                    "dimension": "functional_correctness",
                    "position": "测试用例未通过",
                    "description": f"通过 {exec_result['passed_cases']}/{exec_result['total_cases']} 个测试用例",
                    "severity": "high"
                })
            
            tool_result["tool_feedback"] = json.dumps(exec_result, ensure_ascii=False)

        # 文案/通用场景：调用事实校验工具
        elif self.task_type in ["copywriting", "general"] and "fact_search" in self.tools:
            search_result = self.tools["fact_search"].run(content)
            
            if search_result.get("has_fact_error", False):
                for issue in search_result.get("fact_issues", []):
                    tool_result["tool_issues"].append({
                        "dimension": "fact_accuracy",
                        "position": issue.get("position", "未知位置"),
                        "description": issue.get("description", ""),
                        "correct_info": issue.get("correct_info", ""),
                        "severity": "high"
                    })
            
            tool_result["tool_feedback"] = json.dumps(search_result, ensure_ascii=False)
        
        return tool_result

    def _merge_evaluation(
        self, 
        model_eval: Dict, 
        tool_result: Dict
    ) -> Dict:
        """
        融合模型自评结果与工具客观反馈
        工具发现的问题优先级更高，强制加入问题清单
        """
        merged = model_eval.copy()
        
        # 合并问题清单
        tool_issues = tool_result.get("tool_issues", [])
        if tool_issues:
            # 工具问题标记为高优先级
            for issue in tool_issues:
                issue["source"] = "external_tool"
                merged["issues"].append(issue)
                merged["revision_suggestions"].append(
                    f"根据工具反馈修复：{issue['description']}"
                )
            
            # 客观错误直接拉低评分
            penalty = len(tool_issues) * 1.5
            merged["overall_score"] = max(0, merged["overall_score"] - penalty)
            merged["is_qualified"] = merged["overall_score"] >= self.quality_threshold
        
        # 附加工具原始反馈
        merged["tool_feedback"] = tool_result.get("tool_feedback", "")
        return merged

    def _evaluate(self, task: str, current_output: str, **kwargs) -> Dict:
        """
        重写评估方法：先模型自评，再工具校验，最后融合结果
        """
        # 1. 模型自评（根据任务类型选择对应评估模板）
        if self.task_type == "code":
            model_eval = self._evaluate_code(task, current_output)
        elif self.task_type == "copywriting":
            model_eval = self._evaluate_copywriting(task, current_output)
        else:
            model_eval = super()._evaluate(task, current_output)
        
        # 2. 外部工具客观校验
        tool_result = self._tool_validate(current_output, **kwargs)
        
        # 3. 融合结果
        merged_eval = self._merge_evaluation(model_eval, tool_result)
        return merged_eval

    # ========== 代码场景专用评估 ==========
    def _evaluate_code(self, task: str, code: str) -> Dict:
        prompt = f"""【待评估代码】
{code}

【代码功能要求】
{task}

【评估维度（每项0-10分）】
1. functional_correctness：功能逻辑是否正确，是否实现需求
2. code_quality：可读性、规范度、注释、命名、结构设计
3. robustness：异常处理、边界条件、输入校验完备性
4. performance：时间/空间复杂度合理性
5. compliance：无安全漏洞、无恶意代码

输出标准JSON：
- overall_score: 综合评分
- dimension_scores: 各维度分数字典
- issues: 问题数组，每项含dimension/position/description
- revision_suggestions: 对应修改建议
- is_qualified: overall_score >= {self.quality_threshold}

仅输出纯JSON。"""
        
        response = self.llm_call(self.system_prompt + "\n\n" + prompt)
        return self._extract_json(response)

    # ========== 文案润色场景专用评估 ==========
    def _evaluate_copywriting(self, task: str, content: str) -> Dict:
        style = self.copywriting_config["target_style"]
        audience = self.copywriting_config["target_audience"]
        purpose = self.copywriting_config["core_purpose"]
        
        prompt = f"""【待评估文案】
{content}

【文案原始需求】
{task}

【文案目标配置】
- 目标文风：{style}
- 目标受众：{audience}
- 核心目的：{purpose}

【评估维度（每项0-10分）】
1. task_alignment：是否完全满足需求、字数合规、信息无遗漏
2. style_consistency：文风是否统一、适配目标受众与场景
3. expression_refinement：表达是否精炼流畅、无冗余、用词精准
4. rhetorical_effect：感染力、说服力、节奏感是否达标
5. fact_accuracy：事实、数据、引用是否准确
6. compliance：合规无风险、无敏感内容

输出标准JSON：
- overall_score: 综合评分
- dimension_scores: 各维度分数字典
- issues: 问题数组，每项含dimension/position/description
- revision_suggestions: 对应修改建议
- is_qualified: overall_score >= {self.quality_threshold}

仅输出纯JSON。"""
        
        response = self.llm_call(self.system_prompt + "\n\n" + prompt)
        return self._extract_json(response)

    # ========== 文案润色专用修正提示 ==========
    def _revise(self, task: str, current_output: str, evaluation: Dict) -> str:
        if self.task_type != "copywriting":
            return super()._revise(task, current_output, evaluation)
        
        eval_str = json.dumps(evaluation, ensure_ascii=False, indent=2)
        prompt = f"""【原始文案需求】
{task}

【当前版本文案】
{current_output}

【评估与反馈报告（含工具校验结果）】
{eval_str}

【润色修正规则】
1. 严格保留原文核心信息、观点、数据，只优化表达，不篡改原意
2. 优先修复工具反馈的事实错误，再优化表达与文风
3. 保持目标文风：{self.copywriting_config['target_style']}
4. 修正后全文输出，保持结构连贯，禁止只输出修改片段
5. 不得引入新的事实错误与合规风险

请直接输出修正后的完整文案。"""
        
        return self.llm_call(self.system_prompt + "\n\n" + prompt)

    def run(self, task: str, **kwargs) -> Tuple[str, Dict]:
        """
        增强版主循环，支持透传工具参数（如测试用例、校验配置）
        """
        self.trace["task"] = task
        self.trace["task_type"] = self.task_type
        last_score = None
        
        # 第0轮：初始生成 + 评估
        current_output = self._generate_initial(task)
        initial_eval = self._evaluate(task, current_output, **kwargs)
        current_score = initial_eval["overall_score"]
        
        self.trace["iterations"].append({
            "round": 0,
            "output": current_output,
            "evaluation": initial_eval,
            "score": current_score
        })

        # 主循环
        for i in range(1, self.max_iterations + 1):
            converged, reason = self._check_convergence(
                current_score, last_score, i, initial_eval["is_qualified"]
            )
            if converged:
                self.trace["convergence_reason"] = reason
                self.trace["total_iterations"] = i
                break
            
            # 修正 + 重新评估
            revised_output = self._revise(task, current_output, initial_eval)
            new_eval = self._evaluate(task, revised_output, **kwargs)
            new_score = new_eval["overall_score"]
            
            # 防退化：修正后分数下降则回退并终止
            if new_score < current_score - 0.5:
                self.trace["convergence_reason"] = "修正后质量下降，回退到上一版本"
                self.trace["total_iterations"] = i
                break
            
            self.trace["iterations"].append({
                "round": i,
                "output": revised_output,
                "evaluation": new_eval,
                "score": new_score
            })
            
            last_score = current_score
            current_output = revised_output
            current_score = new_score
            initial_eval = new_eval
        
        self.trace["final_output"] = current_output
        return current_output, self.trace
```

## 六、落地关键注意事项
1. **工具调用成本控制**
    - 工具调用会增加延迟与成本，建议仅在第0轮和关键迭代轮调用，后续迭代优先修正已知问题
    - 可设置「仅当模型自评事实维度低于阈值时才调用搜索工具」的触发策略

2. **主客观反馈权重**
    - 工具反馈的客观错误为硬约束，必须修复
    - 模型自评的表达类问题为软优化，可根据迭代次数取舍
    - 文案场景中，事实准确性优先级高于文采表达

3. **安全隔离**
    - 代码执行必须在隔离沙盒中运行，禁止直接在生产环境执行用户生成代码
    - 搜索结果需做来源可信度分级，优先引用权威来源，避免被错误信息污染

4. **文案润色调参建议**
    - 营销类文案：迭代3轮，阈值8.0，侧重修辞效果与感染力
    - 正式公文类：迭代2轮，阈值8.5，侧重严谨性与规范性
    - 科普类文案：迭代2-3轮，强制开启事实校验，侧重准确性与易懂性
